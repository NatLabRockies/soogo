soogo.optimize.cptv module

Coordinate perturbation and target value strategy.

soogo.optimize.cptv.cptv(fun, bounds, maxeval: int, *, surrogateModel: soogo.model.rbf.RbfModel | None = None, acquisitionFunc: soogo.acquisition.coordinate_perturbation.CoordinatePerturbation | None = None, improvementTol: float = 0.001, useLocalSearch: bool = False, disp: bool = False, callback: Callable[[soogo.optimize.result.OptimizeResult], None] | None = None, seed=None) → soogo.optimize.result.OptimizeResult

Minimize a scalar function of one or more variables using the coordinate perturbation and target value strategy.

This is an implementation of the algorithm desribed in [1]. The algorithm uses a sequence of different acquisition functions as follows:

  1. CP step: surrogate_optimization() with acquisitionFunc.

    This step uses a coordinate-perturbation acquisition (CoordinatePerturbation) that adapts the perturbation scale and explores locally around promising points.

  1. TV step: surrogate_optimization() with a TargetValueAcquisition object.

  2. Local step (only when useLocalSearch is True): Runs a local continuous optimization with the true objective using the best point found so far as initial guess.

The stopping criteria of steps 1 and 2 is related to the number of consecutive attempts that fail to improve the best solution by at least improvementTol. The algorithm alternates between steps 1 and 2 until there is a sequence (CP,TV,CP) where the individual steps do not meet the successful improvement tolerance. In that case, the algorithm switches to step 3. When the local step is finished, the algorithm goes back top step 1.

Parameters:

  • fun – The objective function to be minimized.

  • bounds – List with the limits [x_min,x_max] of each direction x in the search space.

  • maxeval (int) – Maximum number of function evaluations.

  • surrogateModel (Optional[RbfModel]) – Surrogate model to be used. If None is provided, a (default: None) RbfModel model with median low-pass filter is used. On exit, if provided, the surrogate model the points used during the optimization.

  • acquisitionFunc (Optional[CoordinatePerturbation]) – Acquisition function for the CP step. If None, (default: None) uses CoordinatePerturbation configured as in Müller (2016).

  • improvementTol (float) – Expected improvement in the global optimum per (default: 0.001) iteration.

  • useLocalSearch (bool) – If True, the algorithm will perform a continuous (default: False) local search when a significant improvement is not found in a sequence of (CP,TV,CP) steps.

  • disp (bool) – Deprecated and ignored. Configure logging instead using (default: False) standard Python logging levels.

  • callback (Optional[Callable[[OptimizeResult], None]]) – If provided, the callback function will be called after (default: None) each iteration with the current optimization result.

  • seed – Seed or random number generator. (default: None)

Return type:

OptimizeResult

Returns:

The optimization result.

References

[1]

Müller, J. MISO: mixed-integer surrogate optimization framework. Optim Eng 17, 177–203 (2016). https://doi.org/10.1007/s11081-015-9281-2

  • callback (Optional[Callable[[OptimizeResult], None]]) – If provided, the callback function will be called (default: None) after each iteration with the current optimization result. The default is None.

soogo.optimize.multistart_msrs module

Multistart LMSRS optimization routine.

soogo.optimize.multistart_msrs.multistart_msrs(fun, bounds, maxeval: int, *, surrogateModel: soogo.model.base.Surrogate | None = None, batchSize: int = 1, disp: bool = False, callback: Callable[[soogo.optimize.result.OptimizeResult], None] | None = None, seed=None) → soogo.optimize.result.OptimizeResult

Minimize a scalar function of one or more variables using a response surface model approach with restarts.

This implementation generalizes the algorithms Multistart LMSRS from [5]. The general algorithm calls surrogate_optimization() successive times until there are no more function evaluations available. The first time surrogate_optimization() is called with the given, if any, trained surrogate model. Other function calls use an empty surrogate model. This is done to enable truly different starting samples each time.

Parameters:

  • fun – The objective function to be minimized.

  • bounds – List with the limits [x_min,x_max] of each direction x in the search space.

  • maxeval (int) – Maximum number of function evaluations.

  • surrogateModel (Optional[Surrogate]) – Surrogate model to be used. Only used as input, not (default: None) updated. If None is provided, surrogate_optimization() will choose a default model.

  • batchSize (int) – Number of new sample points to be generated per iteration. (default: 1)

  • disp (bool) – Deprecated and ignored. Configure logging instead using (default: False) standard Python logging levels.

  • callback (Optional[Callable[[OptimizeResult], None]]) – If provided, the callback function will be called after (default: None) each iteration with the current optimization result.

  • seed – Seed or random number generator. (default: None)

Return type:

OptimizeResult

Returns:

The optimization result.

References

[5]

Rommel G Regis and Christine A Shoemaker. A stochastic radial basis function method for the global optimization of expensive functions. INFORMS Journal on Computing, 19(4):497–509, 2007.

soogo.optimize.surrogate_optimization module

Core surrogate optimization routine.

soogo.optimize.surrogate_optimization.surrogate_optimization(fun, bounds, maxeval: int, *, surrogateModel: soogo.model.base.Surrogate | None = None, acquisitionFunc: soogo.acquisition.base.Acquisition | None = None, batchSize: int = 1, disp: bool = False, callback: Callable[[soogo.optimize.result.OptimizeResult], None] | None = None, seed=None) → soogo.optimize.result.OptimizeResult

Minimize a scalar function of one or more variables using a surrogate model and an acquisition strategy.

This is a more generic implementation of the RBF algorithm described in [8], using multiple ideas from [9] especially in what concerns mixed-integer optimization. Briefly, the implementation works as follows:

1. If a surrogate model or initial sample points are not provided, choose the initial sample using a Symmetric Latin Hypercube design. Evaluate the objective function at the initial sample points.

  1. Repeat 3-8 until there are no function evaluations left.

  2. Update the surrogate model with the last sample.

  3. Acquire a new sample based on the provided acquisition function.

  4. Evaluate the objective function at the new sample.

  5. Update the optimization solution and best function value if needed.

  6. Determine if there is a significant improvement and update counters.

  7. Exit after nFailTol successive failures to improve the minimum.

Mind that, when solving mixed-integer optimization, the algorithm may perform a continuous search whenever a significant improvement is found by updating an integer variable. In the continuous search mode, the algorithm executes step 4 only on continuous variables. The continuous search ends when there are no significant improvements for a number of times as in Müller (2016).

Parameters:

  • fun – The objective function to be minimized.

  • bounds – List with the limits [x_min,x_max] of each direction x in the search space.

  • maxeval (int) – Maximum number of function evaluations.

  • surrogateModel (Optional[Surrogate]) – Surrogate model to be used. If None is provided, a (default: None) RbfModel model with median low-pass filter is used. On exit, if provided, the surrogate model the points used during the optimization.

  • acquisitionFunc (Optional[Acquisition]) – Acquisition function to be used. If None is (default: None) provided, the TargetValueAcquisition is used.

  • batchSize (int) – Number of new sample points to be generated per iteration. (default: 1)

  • improvementTol – Expected improvement in the global optimum per iteration.

  • nSuccTol – Number of consecutive successes before updating the acquisition when necessary. A zero value means there is no need to update the acquisition based no the number of successes.

  • nFailTol – Number of consecutive failures before updating the acquisition when necessary. A zero value means there is no need to update the acquisition based no the number of failures.

  • termination – Termination condition. Possible values: “nFailTol” and None.

  • performContinuousSearch – If True, the algorithm will perform a continuous search when a significant improvement is found by updating an integer variable.

  • disp (bool) – Deprecated and ignored. Configure logging via standard Python (default: False) logging levels instead.

  • callback (Optional[Callable[[OptimizeResult], None]]) – If provided, the callback function will be called after (default: None) each iteration with the current optimization result.

  • seed – Seed or random number generator. (default: None)

Return type:

OptimizeResult

Returns:

The optimization result.

References

[8]

Björkman, M., Holmström, K. Global Optimization of Costly Nonconvex Functions Using Radial Basis Functions. Optimization and Engineering 1, 373–397 (2000). https://doi.org/10.1023/A:1011584207202

[9]

Müller, J. MISO: mixed-integer surrogate optimization framework. Optim Eng 17, 177–203 (2016). https://doi.org/10.1007/s11081-015-9281-2
