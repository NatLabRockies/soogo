Alternated acquisition function that cycles through a list of acquisition functions.

The current acquisition function moves to the next in the list when the current one’s termination criterion is met. To progress through the acquisition functions, the update method must be called after each optimization step. This provides the function with the current optimization state, allowing it to determine if the termination condition has been satisfied.

class soogo.acquisition.base.Acquisition(optimizer=None, mi_optimizer=None, rtol: float = 1e-06, termination: soogo.termination.TerminationCondition | None = None, multi_objective: bool = False, n_max_evals_optimizer: int = 1000) → None

Bases: ABC

Base class for acquisition functions.

This an abstract class. Subclasses must implement the method optimize().

Acquisition functions are strategies to propose new sample points to a surrogate. The acquisition functions here are modeled as objects with the goals of adding states to the learning process. Moreover, this design enables the definition of the optimize() method with a similar API when we compare different acquisition strategies.

Parameters:

  • optimizer – Continuous optimizer to be used for the acquisition (default: None) function. Default is Differential Evolution (DE) from pymoo.

  • mi_optimizer – Mixed-integer optimizer to be used for the acquisition (default: None) function. Default is Genetic Algorithm (MixedVariableGA) from pymoo.

  • rtol (float) – Minimum distance between a candidate point and the (default: 1e-06) previously selected points relative to the domain size.

  • termination (Optional[TerminationCondition]) – Termination condition for the acquisition function. (default: None) Default is None.

  • multi_objective (bool) – Whether the acquisition function is (default: False) multi-objective. Default is False.

  • n_max_evals_optimizer (int) – Maximum number of function evaluations for (default: 1000) the optimizer. Default is 1000.

class soogo.acquisition.coordinate_perturbation.CoordinatePerturbation(sampling_strategy: str = 'dds', sigma: float | soogo.acquisition.coordinate_perturbation.BoundedParameter | None = None, perturbation_strategy='dycors', n_continuous_search: int = 0, **kwargs) → None

Bases: WeightedAcquisition

Weighted selection with coordinate perturbations around promising points.

This acquisition extends WeightedAcquisition by generating candidates via local, truncated-normal perturbations of the best-known points (DDS/DYCORS-style). It adapts the perturbation scale sigma based on recent successes/failures and can enter a continuous-search mode over only continuous coordinates when improvements in integer variables are detected. See [1] for details.

For multi-objective surrogates, the score uses the average predicted value across targets as \(f_s(x)\). See [2] for details.

Parameters:

  • sampling_strategy (str) – Sampling strategy for candidate generation. (default: 'dds') Currently, only "dds" (default) and "dds_uniform" are supported.

  • sigma (Union[float, BoundedParameter, None]) – Initial perturbation scale; can be a float or a (default: None) BoundedParameter. Default is 0.25 in the range [0.0, 0.25].

  • perturbation_strategy –

    Strategy to set the perturbation probability. (default: 'dycors') Options:

    • "dycors" (default). It exponentially decreases the probability

      of perturbing each coordinate as the number of function evaluations according to the rule:

      \[P_i = min(20/d, 1) * \left( 1 - \frac{\log(k+1)}{\log(N)} \right)\]

      where \(d\) is the problem dimension, \(k\) is the current number of function evaluations, and \(N\) is the maximum number of function evaluations.

    • "fixed". Uses a fixed perturbation probability of 1.0 (all

      coordinates perturbed).

    • "random". Uses a random perturbation probability in [0, 1] at each

      iteration.

    Defaults to "dycors".

  • n_continuous_search (int) – Number of iterations to remain in continuous-search mode after an improvement in integer variables.

Obtain endpoints of the Pareto front as described in [4].

For each component i in the target space, this algorithm solves a cheap auxiliary optimization problem to minimize the i-th component of the trained surrogate model. Points that are too close to each other and to training sample points are eliminated. If all points were to be eliminated, consider the whole variable domain and sample at the point that maximizes the minimum distance to training sample points.

soogo.acquisition.minimize_surrogate module

Minimize surrogate acquisition function using multi-level single-linkage.

class soogo.acquisition.minimize_surrogate.MinimizeSurrogate(pool_size: int, rtol: float = 0.001, seed=None, **kwargs) → None

Bases: Acquisition

Obtain sample points that are local minima of the surrogate model.

This implementation is based on the one of MISO-MS used in the paper [8]. The original method, Multi-level Single-Linkage, was described in [9]. In each iteration, the algorithm generates a pool of candidates and select the best candidates (lowest predicted value) that are far enough from each other. The number of candidates chosen as well as the distance threshold vary with each iteration. The hypothesis is that the successful candidates each belong to a different region in the space, which may contain a local minimum, and those regions cover the whole search space. In the sequence, the algorithm runs multiple local minimization procedures using the successful candidates as local guesses. The results of the minimization are collected for the new sample.

Parameters:

  • pool_size (int) – Number of uniform candidates generated each iteration.

  • rtol (float) – Minimum distance between a candidate and already selected (default: 0.001) points, relative to the domain size. Default is 1e-3.

  • seed – Seed or random number generator. (default: None)

pool_size

Number of uniform candidates generated each iteration.

rng

Random engine used to draw candidates.

References

[8]

Müller, J. MISO: mixed-integer surrogate optimization framework. Optim Eng 17, 177–203 (2016). https://doi.org/10.1007/s11081-015-9281-2

[9]

Rinnooy Kan, A.H.G., Timmer, G.T. Stochastic global optimization methods part II: Multi level methods. Mathematical Programming 39, 57–78 (1987). https://doi.org/10.1007/BF02592071

soogo.acquisition.pareto_front module

Pareto front acquisition functions for multi-objective optimization.

class soogo.acquisition.pareto_front.ParetoFront(optimizer=None, mi_optimizer=None, oldTV=(), seed=None, **kwargs) → None

Bases: Acquisition

Obtain sample points that fill gaps in the Pareto front from [10].

The algorithm proceeds as follows to find each new point:

  1. Find a target value \(\tau\) that should fill a gap in the Pareto front. Make sure to use a target value that wasn’t used before.

  2. Solve a multi-objective optimization problem that minimizes \(\|s_i(x)-\tau\|\) for all \(x\) in the search space, where \(s_i(x)\) is the i-th target value predicted by the surrogate for \(x\).

  3. If a Pareto-optimal solution was found for the problem above, chooses the point that minimizes the L1 distance to \(\tau\) to be part of the new sample.

Parameters:

soogo.acquisition.target_value_acquisition module

Target value acquisition function for RBF surrogate optimization.

class soogo.acquisition.target_value_acquisition.TargetValueAcquisition(cycleLength: int = 6, seed=None, **kwargs) → None

Bases: Acquisition

Target value acquisition function for the RBF model based on [11], [12], and [13].

Every iteration of the algorithm sequentially chooses a number from 0 to cycleLength + 1 (inclusive) and runs one of the procedures:

  • Inf-step (0): Selects a sample point that minimizes the \(\mu\) measure, i.e., mu_measure(). The point selected is the farthest from the current sample using the kernel measure.

  • Global search (1 to cycleLength): Minimizes the product of \(\mu\) measure by the distance to a target value. The target value is based on the distance to the current minimum of the surrogate. The described measure is known as the ‘bumpiness measure’.

  • Local search (cycleLength + 1): Minimizes the bumpiness measure with a target value equal to the current minimum of the surrogate. If the current minimum is already represented by the training points of the surrogate, do a global search with a target value slightly smaller than the current minimum.

After each sample point is chosen we verify how close it is from the current sample. If it is too close, we replace it by a random point in the domain drawn from an uniform distribution. This is strategy was proposed in [14].

Parameters:

  • cycleLength (int) – Length of the global search cycle. Stored in (default: 6) cycleLength.

  • seed – Seed for random number generator. (default: None)

cycleLength

Length of the global search cycle to be used in optimize().

_cycle

Internal counter of cycles. The value to be used in the next call of optimize().

rng

Random number generator.

static bumpiness_measure(surrogate: soogo.model.rbf.RbfModel, x: numpy.ndarray, target, target_range=1.0)

Compute the bumpiness of the surrogate model.

The bumpiness measure \(g_y\) was first defined by Gutmann (2001) with suggestions of usage for global optimization with RBF functions. Gutmann notes that \(g_y(x)\) tends to infinity when \(x\) tends to a training point of the surrogate, and so they use \(-1/g_y(x)\) for the minimization problem. Björkman and Holmström use \(-\log(1/g_y(x))\), which is the same as minimizing \(\log(g_y(x))\), to avoid a flat minimum. This option seems to slow down convergence rates for \(g_y(x)\) in [0,1] since it increases distances in that range.

The present implementation uses genetic algorithms by default, so there is no point in trying to make \(g_y\) smoother.

Parameters:

  • surrogate (RbfModel) – RBF surrogate model.

  • x (ndarray) – Possible point to be added to the surrogate model.

  • target – Target value.

  • target_range – Known range in the target space. Used to scale (default: 1.0) the function values and avoid overflow.

soogo.acquisition.utils.weighted_score(sx, dx, weight: float, sx_min: float = 0.0, sx_max: float = 1.0, dx_max: float = 1.0)

Computes the weighted score from the predicted value of the surrogate model at a point and minimum distance from the point to the set of previously selected evaluation points.

The score is

\[w \frac{s(x)-s_{min}}{s_{max}-s_{min}} + (1-w) \frac{d_{max}-d(x,X)}{d_{max}},\]

where \(w\) is a weight; \(s(x)\) is the value for the surrogate model on \(x\); \(d(x,X)\) is the minimum distance between \(x\) and the previously selected evaluation points; \(s_{min}\) and \(s_{max}\) are the minimum and maximum values of the surrogate model, respectively; and \(d_{max}\) is the maximum distance between a candidate point and the set \(X\) of previously selected evaluation points.

In case \(s_{max} = s_{min}\), the score is computed as

\[\frac{d_{max}-d(x,X)}{d_{max}}.\]
Parameters:

  • sx – Function value(s) \(s(x)\).

  • dx – Distance(s) between candidate(s) and the set X.

  • weight (float) – Weight \(w\).

  • sx_min (float) – Minimum value of the surrogate model. (default: 0.0)

  • sx_max (float) – Maximum value of the surrogate model. (default: 1.0)

  • dx_max (float) – Maximum distance between a candidate point and the set X. (default: 1.0)

class soogo.acquisition.weighted_acquisition.WeightedAcquisition(pool_size: int = 100, weightpattern=None, seed=None, **kwargs) → None

Bases: Acquisition

Select candidates by minimizing a weighted score that balances surrogate value and distance to existing samples.

The weighted score is \(w f_s(x) + (1-w)(-d_s(x))\), where \(f_s(x)\) is the surrogate-predicted value at \(x\) and \(d_s(x)\) is the distance from \(x\) to its closest neighbor in the current sample. Both components are scaled to [0, 1] over the candidate pool. Candidates are generated by a space-filling sampler and the best points under the weighted score are selected. See [15].

This acquisition also supports multi-objective surrogates by scoring with the average predicted target value across objectives. See [16].

Parameters:

  • pool_size (int) – Number of candidates generated per call.

  • weightpattern (float|sequence) – Weight(s) \(w\) used in the score. The list is rotated after each call so subsequent selections consider different trade-offs. Defaults to [0.2, 0.4, 0.6, 0.9, 0.95, 1].

  • seed – Seed or random number generator. (default: None)

rng

Random number generator.

weightpattern

Circular list of weights used in the score; rotated after selections.

pool_size

Number of candidates generated per optimize() call.
